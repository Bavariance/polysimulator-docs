# Virtual → Live Migration

PolySimulator is designed so your entire bot codebase works unchanged when switching from virtual (paper) trading to live (real-money) trading on Polymarket.

---

## Comparison

| Aspect | Virtual Mode | Live Mode |
|--------|-------------|-----------|
| Money at risk | None (simulated $1,000) | Real funds |
| Order execution | Local ledger | Polymarket CLOB |
| Market data | Real prices from Polymarket | Real prices from Polymarket |
| API surface | Full HFT API v1 | Same endpoints |
| Settlement | Automatic on resolution | Polymarket blockchain |

<Warning>
  Live mode executes real trades with real money on Polymarket. Ensure your strategy is thoroughly tested in virtual mode before migrating.
</Warning>

## Migration Steps

<Steps>
  <Step title="Test thoroughly in virtual mode">
    Run your bot through multiple market cycles. Verify:
    - Order placement works correctly
    - Error handling covers all edge cases
    - P&L tracking is accurate
    - WebSocket reconnection is stable
  </Step>

  <Step title="Get Polymarket API credentials">
    Obtain your Polymarket CLOB API credentials:
    - API Key
    - API Secret
    - API Passphrase

    See [Polymarket's documentation](https://docs.polymarket.com) for credential setup.
  </Step>

  <Step title="Update environment variables">
    Change only the base URL and credentials in your bot's environment — **no code changes required**:

    ```bash
    # Virtual mode (PolySimulator) — your current config
    POLYSIM_BASE_URL="https://api.polysimulator.com/v1"
    POLYSIM_API_KEY="ps_live_kJ9mNx2p..."

    # Live mode (Polymarket) — swap these values
    POLYSIM_BASE_URL="https://clob.polymarket.com"
    POLYMARKET_API_KEY="your_polymarket_api_key"
    POLYMARKET_API_SECRET="your_polymarket_api_secret"
    POLYMARKET_API_PASSPHRASE="your_polymarket_api_passphrase"
    ```

    <Info>
      There is no `TRADING_MODE` flag to flip. Migration is achieved by
      pointing your bot at a different base URL and swapping the auth header.
      Your order placement code stays identical.
    </Info>
  </Step>

  <Step title="Use CLOB-compatible endpoint (recommended for migration)">
    The CLOB-compatible endpoint mirrors Polymarket's real CLOB API schema.
    Your bot code stays identical — only the base URL and auth header change:

    <CodeGroup>
    ```python Virtual (PolySimulator)
    import requests, os

    BASE_URL = os.environ["POLYSIM_BASE_URL"]  # https://api.polysimulator.com/v1
    headers = {"X-API-Key": os.environ["POLYSIM_API_KEY"]}

    order = requests.post(f"{BASE_URL}/clob/order", headers=headers, json={
        "token_id": "71321045679252...",
        "side": "BUY",
        "price": "0.65",
        "size": "10",
        "order_type": "GTC",
    })
    order.raise_for_status()
    print(order.json())
    ```

    ```python Live (Polymarket)
    import requests, os

    # Only these two lines change:
    BASE_URL = "https://clob.polymarket.com"
    headers = {"Authorization": f"Bearer {os.environ['POLYMARKET_JWT']}"}

    order = requests.post(f"{BASE_URL}/order", headers=headers, json={
        "token_id": "71321045679252...",
        "side": "BUY",
        "price": "0.65",
        "size": "10",
        "order_type": "GTC",
    })
    order.raise_for_status()
    print(order.json())
    ```
    </CodeGroup>

    <Info>
      **Auth difference**: PolySimulator uses `X-API-Key` header.
      Polymarket uses `Authorization: Bearer <jwt>`. This is the only
      header change needed.
    </Info>
  </Step>

  <Step title="Verify with small orders">
    Start with minimal order sizes to verify live execution:

    ```python
    import requests, os

    BASE_URL = os.environ["POLYSIM_BASE_URL"]
    API_KEY = os.environ["POLYSIM_API_KEY"]
    headers = {"X-API-Key": API_KEY, "Content-Type": "application/json"}

    # 1. Place a small test order
    resp = requests.post(
        f"{BASE_URL}/orders",
        headers=headers,
        json={
            "market_id": "0xabc123...",
            "side": "BUY",
            "outcome": "Yes",
            "quantity": "1",
            "order_type": "market",
        },
    )
    resp.raise_for_status()
    result = resp.json()
    print(f"Order {result['order_id']}: {result['status']} @ {result['price']}")

    # 2. Verify the position was created
    positions = requests.get(
        f"{BASE_URL}/account/positions",
        headers=headers,
        params={"status": "OPEN"},
    ).json()
    print(f"Open positions: {len(positions)}")

    # 3. Check balance impact
    balance = requests.get(
        f"{BASE_URL}/account/balance", headers=headers
    ).json()
    print(f"Balance: ${balance['balance']} | Total: ${balance['total_value']}")
    ```
  </Step>
</Steps>

## CLOB Compatibility Layer

The `POST /v1/clob/order` endpoint accepts the same request body as Polymarket's `POST /order`:

| Field | Type | Description |
|-------|------|-------------|
| `token_id` | string | CLOB outcome token ID |
| `side` | string | `BUY` or `SELL` (uppercase) |
| `price` | string | Limit price as decimal string ("0.01"–"0.99") |
| `size` | string | Number of shares as decimal string |
| `order_type` | string | `GTC` (default), `FOK`, `IOC`, `GTD` |

<Warning>
  All numeric fields (`price`, `size`) must be **strings** (e.g., `"0.65"` not `0.65`).
  `side` must be **uppercase** (`"BUY"` not `"buy"`).
  `order_type` must be a CLOB type (`"GTC"`, `"FOK"`, `"IOC"`, `"GTD"`) — not `"market"`.
</Warning>

See [CLOB Compatibility](/concepts/clob-compatibility) for full schema details and response format.

## Rollback

To return to virtual mode, change your bot's environment variables back:

```bash
POLYSIM_BASE_URL="https://api.polysimulator.com/v1"
POLYSIM_API_KEY="ps_live_kJ9mNx2p..."
# Remove or comment out Polymarket credentials
```

Restart your bot and it resumes paper trading with the simulated balance.
